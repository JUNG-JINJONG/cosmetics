## [2026-04-16] AI 비즈니스 지능화 및 정산 워크플로우 최적화

### 1. AI 비즈니스 코파일럿(Business Copilot) 구축
*   **지능형 데이터 엔진**: Google Gemini AI를 탑재하여 매출, 프로젝트 현황, 정산 대기액 등 실제 DB 데이터를 분석하여 자연어로 답변하는 전용 에이전트 개발.
*   **프리미엄 비즈니스 대시보드**: `/business/copilot` 페이지를 신설하고, 채팅 인터페이스와 실시간 KPI 보드(매출, 활성 프로젝트, 미결제액)가 결합된 UI 구현.
*   **Context-Aware AI**: DB 조인(Join)을 통해 "이번 달 주문한 화장품 리스트"와 같은 구체적인 데이터 추출 능력을 갖춘 지능형 분석 로직 완성.

### 2. 정산(Settlement) 시스템 자동화 및 MVP 최적화
*   **프로세스 간소화**: 제조사의 편의를 위해 '배송 완료' 처리 즉시 해당 주문의 정산 데이터를 '정산 확정' 상태로 자동 전환하는 트리거 로직 구현.
*   **권한별 정산 대시보드**: 바이어는 지불할 금액, 제조사는 수령할 순액(세금, 수수료 제외)을 명확히 확인할 수 있도록 필터링 및 상세 내역(Settlement Details) 화면 제공.

### 3. 발주 및 배송 관리 고도화
*   **데이터 표준화**: `order_status_code` 중심의 공통 코드 체계 확립 및 마이그레이션 완료.
*   **물류 추적**: 송장 등록 및 배송 상태 가시화 UI(Factory & Logistics) 고도화.

### 4. 차기 마일스톤: 클라우드 배포(Cloud Deployment) 준비
*   **로컬 개발 완료 선언**: 2026-04-16 기준, 플랫폼의 핵심 거래 프로세스 및 쇼룸 기능이 로컬 환경에서 완벽하게 검증되었습니다.
*   **배포 로드맵 수립**: 내일(2026-04-17)부터 Docker 컨테이너라이징, HTTPS 보안 인증서(SSL) 적용, 그리고 클라우드 데이터베이스(RDS/Postgres) 이관 작업을 포함한 실서버 배포 공정을 본격 시작할 예정입니다.
*   **사용자 경험 개선**: 네비게이션 바(Navbar)에 'AI 비즈니스' 신규 메뉴를 추가하고, AI 컨설턴트와의 UI 일관성을 확보하여 플랫폼 완성도 제고.

## 📁 프로젝트 폴더 구조 가이드

## ⚠️ 작업 가이드 및 교훈 (Lessons Learned)
*   **단계별 작업 및 승인 원칙 (DB 스키마 변경 시)**: DB 스키마 -> **데이터 작업(I/U/D)** -> 백엔드 모델 -> 서비스 로직 ->
    API 라우터 -> 프론트엔드 순으로 단계를 나누어 작업합니다. **각 단계가 완료될 때마다 정부장에게 작업 결과를 보고하고,
    명시적인 확인을 받은 후 다음 단계로 넘어가는 것을 원칙으로 합니다.** (AI 환각 및 데이터 불일치 방지)
*   **데이터 무결성 및 표준**: 모든 상태 정보는 문자열이 아닌 `category` 테이블의 공통 코드(`_status_code`)를 참조하도록 하며, **공통 코드 데이터 삽입 시 반드시 `design/sql-convention.md`의 코드 명명 및 생성 규칙을 준수합니다.**
*   **프론트엔드 개발 전 검증**: 새로운 화면(UI) 구현을 시작하기 전, **해당 화면에 필요한 데이터를 제공하는 DB 테이블 및 컬럼이 실제로 존재하는지 반드시 먼저 확인합니다.** (데이터 누락으로 인한 중복 작업 방지)
*   **라우팅 표준**: 동적 파라미터 대신 쿼리 스트링을 사용하여 경로 충돌(422 Error)을 원천 차단함.

### 📁 프로젝트 폴더 구조 가이드 (Directory Structure Guide)
*작업 시 모든 신규 파일은 용도에 따라 아래 지정된 폴더에 생성해야 합니다.*

*   **`backend/`**: FastAPI 애플리케이션 소스 코드 및 환경 설정.
    *   `app/api/`: 도메인별 API 라우터 (auth, workflow, business, ai_rd 등).
    *   `app/models/`: SQLAlchemy 데이터베이스 모델.
    *   `app/schemas/`: Pydantic 데이터 검증 스키마.
    *   `app/services/`: 비즈니스 로직 및 외부 연동(AI, Color 등) 서비스.
*   **`design/`**: 프로젝트 기획, 설계 및 표준 정의 문서.
    *   `function-summary.md`: 전체 기능 명세서.
    *   `db-schema.sql`: 최신 통합 데이터베이스 스키마 정의.
    *   `sql-convention.md`: SQL 및 데이터 설계 표준 가이드.
    *   `function_api_list.md`: API 현황 및 상세 명세.
    *   `tech_stack.md`: 기술 스택 및 개발 환경 정의.
*   **`sql/`**: DB 운영 및 마이그레이션 관련 SQL 스크립트.
    *   상태 코드 추가, 데이터 보정, 마이그레이션 전용 `.sql` 파일들.
    *   `backup/`: DB 백업 및 데이터 덤프 파일.
*   **`scripts/`**: 데이터 분석, 초기 데이터 삽입(Seed), AI 테스트용 외부 파이썬 스크립트.
*   **`data/`**: 제품 데이터셋, 원료 리스트 등 로직에 활용되는 원본 데이터 자산.
*   **Root (`./`)**: `work-history.md` (작업 이력), `.env` 등 핵심 설정 파일.
